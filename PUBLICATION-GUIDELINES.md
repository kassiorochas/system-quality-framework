# Publicação Segura de Documentação Técnica

## Nunca publicar
- nome real de cliente
- nome real da empresa
- URL de DEV, UAT ou PROD
- domínio interno
- CPF, telefone, e-mail, token ou id sensível
- prints com informações identificáveis
- código comercializável
- segredo de negócio

## Sempre substituir por nomes genéricos
- Backoffice System
- User Platform
- Core Service
- Payment Provider
- External CRM
- Betting Platform
- Admin Portal

## Quando usar imagem
Antes de publicar:
1. conferir se não há dado pessoal
2. conferir se não há domínio real
3. conferir se não há ids internos
4. borrar ou recortar o que for sensível

## Quando usar código
Só publicar:
- trechos genéricos
- estrutura
- raciocínio
- pseudocódigo
- snippets sem contexto sensível

## Regra prática
Se o conteúdo ajudar alguém a copiar seu produto, descobrir o cliente ou entender demais do sistema real, não publique.
